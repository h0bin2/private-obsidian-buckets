# 캔버스 문서 참조 규칙

## 사용 범위
- 시스템 아키텍처 설명 문서
- 컴포넌트 관계도/흐름도 기반 설계 문서
- 운영/배포 관점 구조 정리 문서

## 저장 위치 (필수)
- 아키텍처 캔버스 문서는 반드시 `PROJECT/<순번_프로젝트명>/docs/` 아래에 저장한다.
- 권장 파일명: `Architecture.canvas`
- 경로 예시: `PROJECT/01_결제고도화/docs/Architecture.canvas`

## 권장 구조
- 목적과 범위
- 시스템 컨텍스트 (외부 시스템 포함)
- 핵심 컴포넌트
- 데이터 흐름
- 인터페이스 (API/이벤트)
- 배포 및 운영 고려사항
- 리스크 및 미결정 이슈

## 작성 규칙
- 컴포넌트 이름은 문서 내에서 일관되게 사용한다.
- 흐름은 입력 -> 처리 -> 저장/전달 순서로 기술한다.
- 외부 의존성은 경계와 책임을 분리해 적는다.
- 다이어그램이 있으면 텍스트 설명을 반드시 함께 둔다.

## 품질 체크
- 누가 봐도 시스템 경계가 명확한가?
- 데이터 흐름이 누락 없이 연결되는가?
- 운영 시 장애 지점과 대응 포인트가 명시됐는가?
