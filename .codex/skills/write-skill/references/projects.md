# 프로젝트 생성 참조 규칙

## 사용 범위
- 신규 프로젝트를 시작할 때 기본 폴더 구조와 문서를 생성한다.
- 여러 프로젝트를 동일한 규칙으로 수평 확장할 때 사용한다.

## 이름 규칙
- 프로젝트 폴더는 반드시 `PROJECT/<순번_프로젝트명>/` 형식을 사용한다.
- `순번`은 2자리 숫자 증가값을 사용한다. (예: `01`, `02`, `03`)
- `프로젝트명`은 영어 kebab-case를 사용한다. (예: `paper-crawling-ai-insights`)
- 예시: `PROJECT/02_literature-qa-pipeline/`

## 기본 폴더 구조
- `PROJECT/<순번_프로젝트명>/docs/`
- 필요 시 하위 폴더를 추가하되, `docs/`는 항상 포함한다.

## 기본 문서 생성 규칙
- `PROJECT/<순번_프로젝트명>/docs/PRD.md`
- `PROJECT/<순번_프로젝트명>/docs/Architecture.canvas`
- 문서 본문은 한글로 작성하되 프로젝트명/경로는 영어 규칙을 유지한다.

## 확장 규칙
- 기존 프로젝트와 동일한 템플릿/섹션 구조를 재사용한다.
- 공통 정책 변경 시 모든 프로젝트 `docs/`에 반영 가능한 형태로 작성한다.
- 프로젝트 간 중복 규칙은 스킬 references로 중앙화한다.

## 인코딩 규칙
- `*.md`, `*.canvas`, `*.yaml`은 UTF-8(BOM 없음)으로 저장한다.
