# PRD - private-obsidian-buckets

## 1) 프로젝트 개요
- 프로젝트명: private-obsidian-buckets
- 목적: Obsidian 볼트 내 문서(`*.md`)와 설정(`.obsidian/**`)의 생성/수정/삭제 이력을 GitHub로 관리한다.
- 핵심 가치: 개인 지식 자산의 버전 관리, 변경 이력 추적, 복구 가능성 확보, 장치 간 동기화 기반 마련.

## 2) 문제 정의
- Obsidian 문서와 설정 변경이 누적되지만, 히스토리 추적/복구가 어렵다.
- 실수 삭제/수정 시 시점 복원이 어렵다.
- 볼트 구조와 설정 변화가 축적되어도 관리 기준이 없다.

## 3) 목표 (Goals)
- 모든 문서/설정 변경을 일관된 Git 워크플로우로 기록한다.
- 최소 1일 1회 이상 커밋 또는 변경 시 즉시 커밋으로 최신 상태를 유지한다.
- 파일 삭제/이동 이력까지 추적 가능하도록 한다.

## 4) 비목표 (Non-goals)
- 협업용 대규모 코드 저장소 운영 최적화.
- 빌드/배포 중심 CI/CD 파이프라인 구축.
- Obsidian 플러그인 개발 자체.

## 5) 사용자 시나리오
- 사용자는 새 노트를 작성하고, 즉시 커밋하여 이력을 남긴다.
- 사용자는 기존 노트를 편집하고 커밋 메시지로 변경 의도를 기록한다.
- 사용자는 노트를 삭제한 뒤 커밋하여 삭제 이력을 보존한다.
- 사용자는 `.obsidian` 설정을 변경하고 커밋하여 환경 재현성을 확보한다.

## 6) 기능 요구사항
- FR-1: `*.md` 파일 생성/수정/삭제를 추적해야 한다.
- FR-2: `.obsidian/**` 하위 설정 파일을 추적해야 한다.
- FR-3: 변경 사항을 `main` 브랜치에 푸시할 수 있어야 한다.
- FR-4: 커밋 메시지 컨벤션을 정의해야 한다. (예: `docs:`, `chore(obsidian):`)
- FR-5: 신규 장치에서 `git clone` 후 Obsidian 볼트가 정상 열려야 한다.

## 7) 비기능 요구사항
- NFR-1: 민감정보 유출 방지를 위해 추적 대상/제외 대상을 명확히 정의해야 한다.
- NFR-2: 저장소 복구 절차가 단순해야 한다. (clone 후 바로 사용 가능)
- NFR-3: 커밋/푸시 절차는 1~3개 명령으로 수행 가능해야 한다.

## 8) 추적 범위/정책
- 기본 포함: `**/*.md`, `.obsidian/**`
- 기본 제외(권장): 대용량 첨부파일, 로컬 캐시, 민감정보 파일
- 저장소 루트에 `.gitignore`를 두고 제외 정책을 문서화한다.

## 9) 운영 정책
- 브랜치 전략: 기본 `main` 단일 브랜치
- 커밋 정책: 의미 단위 커밋 (문서 주제/설정 변경 단위)
- 커밋 메시지 형식: 국제 표준 `Conventional Commits` 사용 (`<type>(<scope>): <subject>`)
- 커밋 메시지 언어: 제목/본문 모두 한글 작성 (단, `type`/`scope` 키워드는 표준 영문 유지)
- 백업 정책: 원격(`origin`) 푸시를 정기 수행

## 10) 성공 지표
- 최근 30일 기준 문서 변경 누락 커밋 0건
- 설정 변경 후 복원 실패 0건
- 실수 삭제 파일 복구 성공률 100%

## 11) 리스크 및 대응
- 리스크: 민감 노트/토큰/개인정보가 원격에 업로드될 수 있음
- 대응: `.gitignore` + 사전 점검(`git status`, `git diff --staged`) 후 커밋

## 12) 오픈 이슈 (부족한 내용)
- 공개 저장소인지 비공개 저장소인지 미정 (강력 권장: private)
- 첨부파일(이미지/PDF) 버전관리 방식 미정
- 자동 커밋/자동 푸시 필요 여부 미정
- 충돌 해결 기준(다중 장치 동시 편집) 미정
- 커밋 주기(즉시/일괄)와 메시지 템플릿 미정

## 13) 초기 액션 아이템
- A1: 저장소를 Private로 설정
- A2: `.gitignore` 초안 생성 (`.obsidian/cache`, OS 메타파일 등)
- A3: `README.md`에 사용 절차(작성->status->add->commit->push) 문서화
- A4: 주 1회 저장소 상태 점검 체크리스트 운영

## 14) 지속 개선 루프
- 매주 1회: 누락 커밋/불필요 추적 파일 점검
- 매월 1회: `.gitignore`/커밋 컨벤션 개선
- 분기 1회: 복구 리허설(clone -> Obsidian 오픈 -> 문서 확인)
