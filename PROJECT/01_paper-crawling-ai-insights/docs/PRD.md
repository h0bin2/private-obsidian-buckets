# PRD - paper-crawling-ai-insights

## 1. 배경
- 논문 탐색, 수집, 정리 과정이 수작업 중심이라 시간이 많이 소요된다.
- 핵심 근거를 빠르게 파악하고 실무 적용 가능성을 평가할 수 있는 체계가 필요하다.

## 2. 문제 정의
- 검색한 논문의 메타데이터와 본문 요약이 흩어져 있다.
- 논문별 품질/적용성 판단 기준이 통일되어 있지 않다.
- 재방문 시 기존 검토 이력을 빠르게 찾기 어렵다.

## 3. 목표
- 논문 수집부터 요약/태깅/적용 메모까지 하나의 파이프라인으로 관리한다.
- AI를 사용해 논문 핵심 내용을 일정한 포맷으로 정리한다.
- 논문별 의사결정 근거(채택/보류/기각)를 문서화한다.

## 4. 비목표
- 논문 원문 불법 수집/배포 기능은 다루지 않는다.
- 범용 검색엔진 개발은 범위에서 제외한다.

## 5. 사용자 시나리오
- 사용자는 키워드 기반으로 논문 후보를 수집한다.
- 사용자는 AI 요약 결과를 검토하고 핵심 근거를 보강한다.
- 사용자는 논문별 적용 메모를 작성하고 우선순위를 부여한다.

## 6. 기능 요구사항
- 논문 메타데이터(제목, 저자, 연도, 링크) 저장
- 논문별 표준 요약 템플릿 생성
- 태그/카테고리 기반 분류
- 적용 가능성 평가(예: 높음/중간/낮음) 기록
- 이력 추적을 위한 Git 기반 버전 관리

## 7. 비기능 요구사항
- 문서 포맷 일관성 유지 (`PAPER_YYYY.MM.DD_제목.md`)
- 빠른 재검색이 가능한 폴더/파일 구조
- 프롬프트/요약 기준의 재현 가능성 확보

## 8. 성공 지표
- 주간 논문 처리량(수집 -> 요약 완료) 증가
- 논문별 적용 메모 작성률 90% 이상
- 재검색 시간(대상 논문 찾기) 단축

## 9. 리스크 및 대응
- AI 요약의 사실 왜곡 가능성: 원문 근거 링크 필수 기록
- 과도한 수집으로 인한 정리 지연: 우선순위 규칙 도입

## 10. 오픈 이슈
- 초기 수집 소스 우선순위(arXiv, ACL, IEEE 등)
- 자동 수집 주기 및 실행 방식
- 평가 스코어링 기준 상세화
