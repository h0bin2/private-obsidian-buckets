# private-obsidian-buckets

## 커밋 메시지 규칙
- 국제 표준 `Conventional Commits`를 사용한다.
- 형식: `<type>(<scope>): <subject>`
- `type`은 `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `build`, `ci`, `perf`를 사용한다.
- `subject`와 본문은 한글로 작성한다.
- 예시:
  - `docs: PRD에 커밋 메시지 정책 추가`
  - `chore(obsidian): 워크스페이스 설정 파일 추적`
